## 7. La librer√≠a DronLib

DronLib es la librer√≠a que vamos a usar para **darle ordenes al dron** (tanto al simulador SITL
como al dron real). Es una **librer√≠a en desarrollo** (y por tanto, no exenta de fallos) que

mantenimiento y no es compatible con las versiones m√°s avanzadas del int√©rprete de Python.

La librer√≠a esta implementada en forma de clase **(la clase Dron)** con sus atributos y una
variedad de m√©todos para operar con el dron. La clase con los atributos est√° definida en el
fichero _Dron.py_ y los m√©todos est√°n en los diferentes ficheros de la carpeta _modules_
(connect.py, arm.py, etc.).

Muchos de los m√©todos pueden activarse **de forma bloqueante o de forma no bloqueante**. En
el primer caso, **el control no se devuelve al programa que hace la llamada hasta que la
operaci√≥n ordenada haya acabado**. Si la llamada es no bloqueante entonces **el control se
devuelve inmediatamente** para que el programa pueda hacer otras cosas mientras se realiza la
operaci√≥n.

Un buen ejemplo de m√©todo con estas dos opciones es _takeOff_, que tiene la siguiente cabecera:

```

```

Al llamar a este m√©todo hay que pasarle como par√°metro la **altura de despegue**. Por defecto la
operaci√≥n es **bloqueante**. En el caso de usar la opci√≥n no bloqueante se puede indicar el
nombre de la funci√≥n que queremos que se ejecute cuando la operaci√≥n haya acabado (que
llamamos habitualmente **callback**). Incluso podemos indicar los par√°metros que queremos que
se le pasen a ese callback. **Recuerda que self no es ning√∫n par√°metro**. Simplemente **indica que
este es un m√©todo de una clase** (en este caso, la clase Dron).
